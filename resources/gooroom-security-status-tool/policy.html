정책
권한 정책 정보
브라우저 정책을 확인할 수 있습니다. 브라우저 정책은 접속 사이트를 신뢰/비신뢰 사이트로 구분하고, 브라우저에서 사용할 수 있는 정책들을 제어하는 정책입니다. 구름 브라우저에서 선택하는 정책이 아닌, GPMS에서 내려주는 정책을 반영합니다.


매체 제어 정책
USB 메모리, CD/DVD, 무선인터넷 등 주변기기와 시스템, 네트워크의 사용을 제어합니다. 단말의 [매체 제어 편집기]에서 편집한 내용이 아닌, GPMS에서 내려주는 정책을 반영합니다.


보안
보안 상태 정보 탭으로 들어가면 다음과 같은 3가지 상태를 확인할 수 있습니다. 


보안 상태

신뢰부팅, 실행파일 검증, 운영체제 보호, 매체제어의 상태를 판단하여 안전/취약으로 현재 상태를 나타냅니다. 안전 조치 실행버튼은 [취약] 상태에서 활성화 됩니다. 


활성화된 보안 기능

신뢰부팅, 실행파일 검증, 운영체제 보호, 메체제어가 활성화 되어 있는지 나타냅니다. 정보를 표시하는 기능일 뿐 편집할 수 없습니다. 매체제어 편집기 를 통해 매체제어를 활성화 할 수 있습니다.


단말 보안 정책

계정의 패스워드 변경 주기, 화면보호기 설정 시간, 패키지 추가/삭제 차단 기능의 사용 여부를 나타냅니다.


네트워크
GPMS에서 설정한 네트워크 정책과 방화벽 정책을 보여줍니다.


알림
알림창 표시 조건 (로그 레벨)

 구름 플랫폼은 구름 보안프레임워크가 제공하는 4단계 보안 기술을 제공합니다.
신뢰부팅부트로더 및 커널에 대한 무결성 검증 수행으로, 해커에 의해 시스템이 변조되었을 경우 구름 플랫폼의 부팅을 차단합니다.
운영체제 보호동적 커널 보호 기술로서, 악성코드가 실행중인 커널을 변조할 때, 커널보다 높은 수준(하이퍼바이저)에서 악성행위를 탐지합니다.
실행파일 보호실행파일에 대한 무결성 검증 수행으로, 악성 프로그램이 단말에 유입되었을 경우에도 근본적인 실행을 차단합니다.
브라우저 보호접속처에 따라 신뢰브라우저와 비신뢰브라우저로 격리된 브라우징 환경을 제공하고 차등화된 보안정책을 적용할 수 있습니다. 
알림창 표시 조건은 위의 기술 3가지(1. 신뢰부팅 / 2.실행파일 검증 / 3.운영체제 보호)과 매체제어 기술을 이용하여 로그를 기록합니다. 각 영역의 체크박스는 로그를 기록하는 범위를 나타내며, 구름 시스템 보안 상태 자세히 보기 프로그램을 이용해 수정할 수는 없습니다.


<보안 상태> 중복 알림 허용

보안 상태 위험에 대한 알림이 여러 건일 때, 중복 알림을 허용하거나 1회만 알림을 허용하도록 설정할 수 있습니다.
